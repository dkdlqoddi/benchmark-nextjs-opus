# p11 — Habit tags + tag-filterable home page

## 1) Work summary

Habits can now carry **multiple tags**, and the home page is **filterable by
tag**.

- **Data model.** New `Tag` model (per-user: `@@unique([userId, name])`, `userId`
  FK with cascade) and an implicit many-to-many `Habit <-> Tag` relation (Prisma
  join table `_HabitToTag`, both sides cascade on delete). Migration
  `add_tags`. Tags are scoped per user, so one user's "Health" is a distinct row
  from another's.
- **Editing tags.** The habit form gained a comma-separated **Tags** input. On
  save, the create/update actions `parseTags()` the input (trim, drop blanks,
  de-dupe case-insensitively), validate it (≤10 tags, ≤30 chars each), then
  upsert each tag for the user and `connect`/`set` them on the habit. `update`
  uses `set` so removing a tag from the field detaches it.
- **Filterable home.** The home page reads `?tag=<name>`. It renders a filter
  bar ("All" + each tag that's on an active habit), highlights the active chip
  (`aria-current="page"`), and filters the habit query with
  `tags: { some: { name } }`. Empty results under a filter show a contextual
  "No active habits tagged …" message with a "Show all" link.
- **Tags on cards/detail.** Habit cards and the detail page show tag chips; each
  chip links to `/?tag=<name>` so clicking a tag filters the home page.
- **Seed.** Seeded habits now have tags (Alice: Health, Learning; Bob: Fitness,
  Health, Mindfulness — "Health" shared across both of Bob's habits), so
  filtering is demonstrable.
