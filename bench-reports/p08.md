## 4) Key decisions and reasons

- **Root deliverable kept separate from the report.** Per the task, the analysis
  lives in `ARCHITECTURE.md` at the repo root; `bench-reports/p08.md` only
  summarizes the process. No content was duplicated between them.
- **Documented the *actual* Next.js 16 conventions, not training-lore.** Followed
  `AGENTS.md` and read the bundled docs so the doc correctly reflects async
  `params`/`searchParams`, `force-dynamic` route config + `revalidatePath`
  (rather than Cache Components), and React 19 Server Actions.
- **Explained *why* `force-dynamic` appears only on some pages.** The doc notes
  that param-less DB-backed pages (Home/Stats/Archived) need `force-dynamic` to
  avoid stale prerenders, whereas dynamic-segment pages (`[id]`, `edit`) are
  already per-request because they `await params`. The build output verifies
  this classification.
- **Used fenced ASCII diagrams + tables rather than Mermaid** so the document
  renders identically in any Markdown viewer (no dependency on diagram support).
- **Formatted ARCHITECTURE.md with Prettier.** Since it is not in
  `.prettierignore`, leaving it failing `prettier --check` would violate the
  project's formatting standard; Prettier does not touch fenced code, so the
  diagrams were preserved (only tables/prose were normalized).
- **Ran the full code gates despite a docs-only change.** The CLAUDE.md checklist
  is per-task; running `tsc`/`lint`/`build` confirms (and documents) that the
  repo remains green.
