# p13 — Build the test suite

## 1. Work summary

Added a two-layer automated test suite to HabitLog:

- **Vitest unit tests** (`tests/unit/`, Node environment, `@/` alias mirroring
  `tsconfig`), covering the pure domain modules:
  - **`lib/streak.ts`** — 22 table-driven streak cases plus dedicated
    `computeStats`/`weeklyCompletionRates` tests (27 tests total). Includes all
    ten cases previously asserted by `scripts/verify-streak.ts` **plus** edge
    cases: **year boundary** (Dec→Jan, contiguous and with a gap), **leap day**
    (Feb 29 2028 present, missing, and non-leap Feb 2027), and **target-day
    combinations** (Mon–Fri weekend-skipping, weekends, weekly Thursday cadence,
    grace day, empty mask, all-non-target check-ins).
  - **`lib/date.ts`** — Asia/Seoul **timezone boundary** cases (KST-midnight
    roll-over at 15:00 UTC, year boundary in Seoul vs UTC, UTC-midnight instant),
    plus `shiftKey`/`daysInMonth`/`addMonths`/`buildMonthGrid` leap/rollover
    coverage and fake-timer tests for `getTodayKey`/`currentMonth`.
  - **Zod schemas** — `habitSchema`, `loginSchema`, `signupSchema`, each with
    ≥3 valid and ≥3 invalid inputs (10 / 7 / 7 respectively).
- **Playwright E2E tests** (`tests/e2e/`, headless Chromium):
  - **Full flow**: signup → log out → log in → create habit → check in → view
    stats (asserts the 1-day streak on `/stats`).
  - **Data isolation**: a second user sees an empty home page and gets a **404**
    when navigating directly to the first user's habit URL.
  - Runs against a **production build** (`next build && next start`) on port 3100,
    backed by a **throwaway migrated SQLite DB** provisioned in `global-setup`
    (`prisma migrate deploy` into `prisma/test.db`) — `prisma/dev.db` is never
    touched.
- **Scripts**: added `test` (`vitest run`) and `test:e2e` (`playwright test`);
  both run headless and non-interactively (CI-safe). Removed `verify:streak`.
- **Absorbed & deleted** `scripts/verify-streak.ts` (its cases now live in
  `tests/unit/streak.test.ts`); removed the now-empty `scripts/` directory.
- Updated ignore/lint configs for Playwright output dirs and refreshed
  `ARCHITECTURE.md`'s testing/scripts sections.

All four quality gates (build, tsc, lint) and both test suites pass.

## 4. Key decisions and reasons

- **Read the bundled Next 16 testing guides first** (`node_modules/next/dist/
  docs/01-app/02-guides/testing/{vitest,playwright}.md`, per `AGENTS.md`). They
  confirmed the Vitest+config and Playwright `webServer` patterns, and that
  headless is the CI default. Also confirmed via the CLI docs that `next dev`
  outputs to `.next/dev` while `next build`→`next start` uses `.next`, so E2E's
  production server never conflicts with a dev server.
- **Node environment + manual `@/` alias for Vitest** (no jsdom / RTL /
  plugin-react): every unit target is a pure function or zod schema, so React
  rendering deps are unnecessary. Kept the dependency surface minimal
  (`vitest` only) and aliased `@`→repo root to resolve `@/lib/*` imports.
- **Expected values verified against the real implementation** before asserting:
  ran throwaway `tsx` probes for every streak, date, and schema case so no
  expected value was guessed (e.g. confirmed Zod 4's `z.email()` rejects
  `missing@tld`, and the leap-day/weekend-skip streak counts).
- **E2E DB = `prisma migrate deploy` into an absolute-path throwaway DB.**
  `dev.db` is stale (pre-auth schema) and `migrate reset` is gated by the
  Prisma AI-agent guardrail; `migrate deploy` forward-applies all migrations
  into a fresh file and is not gated, so it works unattended. An absolute path
  keeps the CLI and the running server's Prisma client on the same file; the
  server's `DATABASE_URL` is overridden via `webServer.env` (process env wins
  over `.env` in Next).
- **Production build for E2E** (`next build && next start`) rather than
  `next dev`: more representative and more stable for Server Actions, and makes
  `test:e2e` a single self-contained CI command.
- **Serial run, 1 worker**: the two specs share one SQLite file; serial
  execution avoids write-lock flakiness. Each test uses distinct account emails.
- **Isolation asserted two ways**: intruder's home shows the empty state / zero
  habit cards, and a direct `GET /habits/<owner-id>` returns HTTP **404**
  (the detail page's `findFirst({ id, userId })` + `notFound()`).

## 5. Remaining issues/TODOs

- E2E runs Chromium only. Firefox/WebKit projects could be added but need extra
  browser downloads; Chromium is sufficient for the required flows.
- `test:e2e` rebuilds the app on every run (bulletproof but ~3s of build). A
  prebuilt-server variant could be added for faster local iteration.
- Playwright system browser deps were installed via
  `npx playwright install-deps chromium` (apt, root); a CI image should run the
  same, or `npx playwright install --with-deps chromium`.
- Prisma prints a deprecation notice (`package.json#prisma` → `prisma.config.ts`)
  during `migrate deploy`; harmless and out of scope for this task.
