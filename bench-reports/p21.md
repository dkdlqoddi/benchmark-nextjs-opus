### Security / quality backlog (documented in `AUDIT.md`, p18 — none fixed)
- **M1 — No auth rate limiting.** `loginAction`/`signupAction`/`authorize` have no
  throttling; brute-force/enumeration at network speed. Needs a shared store. _[Medium]_
- **M2 — Account enumeration.** Signup says "email already exists"; login's
  `authorize` is a timing oracle (fast reject for unknown email vs. bcrypt for
  known). _[Medium]_
- **M3 — Unhandled `P2002` on concurrent writes.** Same class as the fixed H2:
  concurrent duplicate signup (`User.email`) and concurrent tag `upsert`
  (`resolveTagIds`) can throw instead of degrading gracefully. _[Medium]_
- **L2 — Minor over-fetching.** `/stats` and `/habits/archived` `findMany` without
  a `select`, pulling all `Habit` columns. _[Low]_
- **L3 — Orphaned tags accumulate.** Tags unlinked from every habit stay in the
  `Tag` table (hidden from the filter). A prune job could clean them. _[Low]_
- **L4 — `AUTH_SECRET` committed in `.env`.** Intentional dev-only convention;
  **must be rotated and untracked for any real deployment**. _[Low]_
- **L5 — Picker grouping semantics.** Color / target-day button groups use a
  `<span>` label rather than `<fieldset><legend>`. _[Low]_

### Tooling / dependencies
- **Prisma config deprecation.** `package.json#prisma` (the `seed` config) is
  deprecated and **removed in Prisma 7** — still emits a warning on every Prisma
  command (verified live). Migrate to `prisma.config.ts` before upgrading. Flagged
  in every report since p01; harmless on Prisma 6. _[Low, but a hard blocker for the Prisma 7 upgrade]_
- **npm audit: 2 moderate advisories.** `next` depends on a vulnerable `postcss`
