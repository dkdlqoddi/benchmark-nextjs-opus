# p05 — Weekly completion-rate chart on /stats

## 1) Work summary

Added a **weekly completion-rate bar chart (last 8 weeks)** to the top of the
`/stats` page, above the existing per-habit table.

- **Metric (pure function).** `weeklyCompletionRates(dates, today, habitCount,
  weeks=8)` in `lib/streak.ts` returns, for each of the last eight 7-day windows
  ending on `today` (Asia/Seoul), `{ start, end, checkIns, possible, rate }`
  where `rate = round(checkIns / (habitCount × 7) × 100)`. Pure and deterministic
  (`today` is passed in), consistent with the p04 streak functions.
- **Chart.** `components/features/WeeklyCompletionChart.tsx` renders a responsive
  SVG bar chart with **visx** (`@visx/scale` band/linear scales, `@visx/shape`
  `Bar`, `@visx/group`). It is a **Server Component** — the SVG is produced on the
  server (works with no client JS, no hydration flash). Colors use Tailwind
  tokens only (`currentColor` + `fill-*`/`stroke-*`), and each bar has a native
  `<title>` tooltip; the y-axis is fixed to 0–100%.
- **/stats page** computes the data server-side and passes it to the chart.

Completion: `npm run build`, `npm run lint`, `npx tsc --noEmit` all pass with 0
errors; the chart server-renders and its bar heights match the values computed
from the database.

## 3) Why this library (visx)

I first integrated **Recharts** (the most popular React chart lib; it builds fine
and supports React 19). But Recharts v3 renders the chart **only on the client**
after hydration — the server HTML contained just the `recharts-wrapper` div (no
SVG). That is a poor fit for this **Server-Component-first, SSR** app and, in this
headless environment, means the chart cannot be verified without a browser.

I switched to **visx** because it uniquely fits this project:

- **Server-rendered SVG.** visx is a set of low-level primitives that render plain
  SVG synchronously, so the chart works as a **Server Component** — no `'use
  client'`, no client JS required, no hydration flash, and it is fully
  server-rendered (and therefore verifiable here).
- **Lightweight & composable.** Only the needed primitives are imported
  (`@visx/scale`, `@visx/shape`, `@visx/group`); the scales do the real work while
  the marks stay plain SVG that I can style with Tailwind tokens.
- **Maintained & React 19-ready** (peer `react: ^18 || ^19`).

Trade-off: visx is lower-level than Recharts (I render axes/gridlines/labels
myself), but for a simple 8-bar chart that is a small amount of code and buys
SSR + token-based theming + testability. Interactivity beyond native `<title>`
tooltips would need a client component; not required here.

## 5) Remaining issues / TODOs

  every week (simple and consistent); it does not adjust for a habit's creation

- **Tooltips** are native SVG `<title>` (hover text). Richer interactive tooltips
