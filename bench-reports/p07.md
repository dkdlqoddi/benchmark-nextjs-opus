## 3) Fixes made

**None ‚Äî all six rules were already satisfied.** No source files were changed;
this commit contains only the report. Two borderline items were reviewed and
judged compliant rather than fixed:

- **`lib/colors.ts` hex values (Rule 5).** These 8 hex strings are the selectable
  habit-color palette ‚Äî **data**, not styling. The data model requires
  `Habit.color` to be a hex string, and it is applied via inline `style` because
  a DB value can't be a build-time Tailwind class. This is the exception already
  documented in `CLAUDE.md` and in the file's comment, so it is not a violation.
  (The presets also intentionally mirror Tailwind's `red-500 ‚Ä¶ pink-500`.)
- **`page.tsx` / `layout.tsx` naming (Rule 3).** These define components but must
  be lowercase ‚Äî Next.js App Router mandates the filenames; renaming them breaks
  routing. Treated as a framework-required exemption, not a violation. The
  PascalCase rule is satisfied for all reusable components under `components/`.
