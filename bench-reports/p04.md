# p04 — Statistics (streaks) + /stats page

## 1) Work summary

- **Pure streak library `lib/streak.ts`.** `currentStreak(dates, today)`,
  `longestStreak(dates)`, and `computeStats(dates, today)` are pure functions —
  `today` is passed in as a `YYYY-MM-DD` key (no `Date.now()`, no I/O), so they
  are deterministic and unit-testable. Rules implemented exactly:
  - a streak is a run of check-ins on consecutive dates;
  - the current streak anchors at **today** if checked in, otherwise at
    **yesterday** (so it is preserved before today's check-in);
  - it is **0** when neither today nor yesterday is checked in.
- **`/stats` page.** Rewrote `app/stats/page.tsx` (Server Component) to show a
  per-habit table of **current streak / longest streak / total check-ins** for
  all active habits, using `computeStats` with `getTodayKey()` (Asia/Seoul). Each
  row links to the habit's calendar.
- **Verification script `scripts/verify-streak.ts`** covers all required cases
  (today = 2026-07-02) and exits non-zero on any failure. Added an
  `npm run verify:streak` script.

Completion: all specified streak cases pass, and `npm run build`,
`npm run lint`, `npx tsc --noEmit` pass with 0 errors.

## 4) Key decisions and reasons

- **`today` as a parameter.** The functions never read the clock; the page/script
  pass a `YYYY-MM-DD` key. This keeps the logic pure and lets the verifier pin
  today = 2026-07-02.
- **String date keys + `shiftKey`.** Consecutive-day checks use a `Set` of keys
  and `shiftKey(key, ±1)` (from `lib/date`, pure, UTC-based), so ordering is a
  plain string compare and there is no timezone coupling inside the math.
- **Anchor logic for current streak.** Anchor at today if present, else yesterday,
  else 0; then walk backwards while consecutive days are present — a direct
  encoding of the three rules.
- **O(n) longest streak.** Only walk forward from run starts (days whose previous
  day is absent), so each date is visited once.
- **Stats scope = active habits** (archived excluded), consistent with the home
  page; each row links to the habit's calendar detail.
